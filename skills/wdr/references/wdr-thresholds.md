# WDR 阈值口径（OLTP 取向，v1 内置常量、无覆盖 flag）

这些是 `wdr.py collect` 的确定性发现阈值默认值（`thresholds.py` 常量）。**严重度由脚本按这些阈值判定，LLM 不得更改。**

| 维度 | 🟡关注 | 🟠告警 | 🔴严重 |
|---|---|---|---|
| 单条 SQL 占 DB time | >30% | >50% | >80% |
| 高 DB time 但 CPU 极低（CPU/elapsed） | <10% → `WDR_TOPSQL_LOW_CPU`(阻塞/睡眠,非资源元凶) | — | — |
| 回滚率（事务总数≥1000 才判） | >5% | >15% | — |
| 死锁数（窗口内） | ≥1 | ≥5 | — |
| 缓存命中率 | <99% | <95% | — |
| 临时文件溢出 | >512MiB | >2GiB | — |
| 等待类倾斜（单一非空闲类占比） | >40% | >60% | — |
| 强制 checkpoint 占比 | >30% | >50% | — |

阈值理由：OLTP 下单条 SQL 占 DB time >50% 即明显热点；回滚率 >5% 多为应用异常；缓存命中 <99% 对 OLTP 已偏低。与 health 阈值口径对齐。
